## 1. 概述

本文档旨在详细描述 ParallelChat V1.2 (MVP) 的各项功能组件。每个组件的规格将从UI构成、功能逻辑和用户交互三个维度进行阐述，作为UI/UX设计和软件开发的直接依据。V1.2版本在前一版的基础上，重点优化了核心交互效率、布局灵活性及新用户引导体验。

## 2. 组件化功能规格

### 2.1. 组件一: 应用主框架 (App Shell)

**PRD关联:** F-001, F-007

**组件描述:**  
应用的 GFX 层容器，定义了整体的布局结构。它是一个典型的三段式布局：左侧为可收起的会话侧边栏，右侧为主工作区，底部为常驻的全局输入栏。

**UI构成:**
- **窗口标题栏:** 显示应用名称 "ParallelChat" 和标准的窗口控制按钮（最小化、最大化、关闭）
- **主容器:** 一个 flex 或 grid 布局容器
  - **左侧插槽 (Slot):** 用于放置 会话侧边栏 组件
  - **右侧主区域:** 一个 flex 容器，垂直布局
    - **顶部插槽:** 用于放置 工作区 组件
    - **底部插槽:** 用于放置 全局输入栏 组件

**功能逻辑与交互:**
- 主框架负责响应窗口大小的变化，并确保内部布局的自适应调整
- 会话侧边栏 的宽度可由用户拖拽其右边框进行调整，最小/最大宽度需要设定阈值（如：最小200px，最大500px）
- 全局输入栏 的高度会根据内容动态调整
- 工作区 的尺寸则填充剩余的可用空间

### 2.2. 组件二: 会话侧边栏 (Session Sidebar)

**PRD关联:** F-007

**组件描述:**  
位于应用主框架左侧，是用户管理和导航所有历史对话的核心区域。

**UI构成:**

**顶部操作区:**
- **[按钮] 新建会话 (+ New Chat):** 视觉上最突出的按钮，用于创建新的空白对话

**会话列表区:**
- 一个可滚动的列表容器
- **[列表项] 会话条目 (Session Item):** 列表中的每一项代表一个独立的会话
  - **状态:** 包含"默认态"和"选中态"（有高亮背景）
  - **内容:** 显示会话标题（默认为首次提问内容）
  - **交互控件 (鼠标悬浮时出现):**
    - **[图标按钮] 重命名 (Edit)**
    - **[图标按钮] 删除 (Delete)**

**功能逻辑与交互:**

**新建会话:**
1. 点击 + 新建会话 按钮
2. 工作区 内的所有 AI视图 清空历史记录，加载对应AI官网的初始页面
3. 此时会话列表不创建新条目，直到用户在 全局输入栏 发送第一条消息后，才在列表顶部生成一个新的 会话条目

**切换会话:**
1. 点击任意一个 会话条目
2. 该条目变为"选中态"
3. 工作区 内的所有 AI视图 同步加载并恢复到该会话对应的历史记录状态

**重命名会话:**
1. 鼠标悬浮于 会话条目 上，点击 重命名 图标
2. 会话标题变为一个可编辑的文本输入框，用户输入新标题后按 Enter 或失焦即可保存

**删除会话 (V1.2 优化):**
1. 鼠标悬浮于 会话条目 上，点击 删除 图标
2. 弹出一个二次确认对话框："您确定要删除此会话吗？此操作无法撤销。"
3. 确认后，该 会话条目 从列表中移除
4. **焦点切换逻辑:** 如果删除的是当前选中的会话，系统将优先选中其上方的会话。若被删除的是列表中的第一项，则自动选中新的第一项（即原先的第二项），确保焦点始终停留在可预测的位置

> **注意:** 所有会话数据（ID、标题、各WebView的URL状态）需本地持久化存储。

### 2.3. 组件三: 工作区 (Workspace)

**PRD关联:** F-001

**组件描述:**  
用户与各个AI进行交互的主要区域，由一个或多个 AI视图 组成，并支持多种布局模式。

**UI构成:**
- **布局容器:** 根据用户选择的模式，可以是 grid 容器 (网格/瀑布流布局) 或 tab 容器 (标签页布局)
- **工作区操作栏:**
  - 位于布局容器的右上角或一个固定的标签栏位置
  - **[图标按钮] 添加AI视图 (+):** 用于触发 添加AI对话框
  - **[图标按钮] 布局切换:** 用于在"网格/瀑布流布局"和"标签页布局"之间切换
- **占位符/引导区:**
  - 当没有任何 AI视图 时，显示引导信息，如："点击右上角的 '+' 按钮，添加您的第一个AI助手"

**功能逻辑与交互:**
- 工作区 负责渲染 设置 中配置好的 AI视图 列表
- 点击 添加AI视图 (+) 按钮，会弹出 添加AI对话框

**布局模式管理 (V1.2 新增):**

**模式一: 网格/瀑布流布局 (Grid/Waterfall Layout - 默认)**
- **描述:** 允许多个 AI视图 在工作区内同时可见，以网格形式排列。适合需要并行对比多个AI回复的场景
- **交互:** 用户可以拖拽 AI视图 之间的间隙（Gutter）来自由调整各自的尺寸，并可拖拽标题栏重新排序

**模式二: 标签页布局 (Tabs Layout)**
- **描述:** 工作区顶部出现一个标签栏，每个 AI视图 对应一个标签页。任何时候只有一个 AI视图 是可见的。适合需要深度聚焦于单个AI进行连续交互的场景
- **交互:** 用户通过点击不同的标签来切换 AI视图，并可拖拽标签重新排序

> **注意:** 布局模式、尺寸、顺序等状态需要被记忆，在应用重启后恢复。

### 2.4. 组件四: AI视图 (AI View)

**PRD关联:** F-001, F-002, F-003

**组件描述:**  
一个独立的容器，用于内嵌单个AI的官方网站，并提供应用层面的控制和状态反馈。

**UI构成:**

**视图标题栏:**
- **[文本] AI名称:** 如 "Gemini", "Kimi"
- **[图标按钮] 关闭 (x):** 用于从工作区移除此视图
- **[状态指示器]:** 一个彩色圆点或高亮边框，用于表示该视图是否会接收广播。此指示器可直接点击交互
- **[文本/图标] 错误提示:** 默认隐藏。当消息发送失败时出现，如 "发送失败"

**内容区:**
- **[WebView]:** 内嵌AI官网的核心组件

**功能逻辑与交互:**

**移除视图:** 点击标题栏的 关闭 按钮，该视图将从工作区中移除。此操作仅影响当前布局，AI配置仍在设置中保留。

**快速切换广播目标 (V1.2 优化):**
- 用户可以直接点击标题栏上的 状态指示器 来快速切换该AI的广播接收状态（激活/禁用）
- 此操作将与 全局输入栏 中对应的AI选择状态双向同步

**独立交互:** 用户可以直接在 WebView 组件内进行所有原生网页操作（滚动、点击、复制等）

**接收广播:** 当 全局输入栏 发起广播时，如果此视图处于广播接收激活状态，它将接收到文本内容，并执行注入的JS自动化脚本来完成输入和发送

**状态反馈:** 脚本执行失败（如找不到输入框），错误提示 将在标题栏显示，并在短暂延迟后消失

> **注意:** 每个 AI视图 拥有独立的会话存储 (Session Partitioning)，确保登录状态互不干扰。

### 2.5. 组件五: 全局输入栏 (Global Input Bar)

**PRD关联:** F-002

**组件描述:**  
位于应用底部的常驻输入区域，是"一次提问，多端响应"核心功能的入口。

**UI构成:**

**AI选择器 (V1.2 动态适配):**
- 此区域的UI形态将根据 工作区 的布局模式动态改变

**输入区:**
- **[多行文本输入框 Textarea]:**
  - 支持多行输入
  - 有清晰的占位提示文本，如："在此输入你的问题，按下 Ctrl+Enter 广播..."

**操作区:**
- **[按钮] 广播发送 (Broadcast):**
  - **状态:** 默认态、加载中态（显示Spinner动画）、禁用态（输入框为空时）
  - 鼠标悬停时，有Tooltip提示完整的快捷键（如 "Cmd+Enter on macOS"）

**功能逻辑与交互:**

**AI选择 (动态适配逻辑):**

**当工作区为"网格/瀑布流布局"时:**
- AI选择器 表现为 **[复选框组 (Checkbox Group)]**
- 用户通过勾选/取消勾选来决定广播目标

**当工作区为"标签页布局"时:**
- AI选择器 表现为 **[紧凑标签组 (Chip Group)]**
- 默认仅当前激活的AI标签被选中，但用户可手动点选其他AI标签，将其加入本次广播

**输入:** 用户在 Textarea 中输入文本。当输入框有内容时，广播发送 按钮变为可用状态。

**发送:**
1. 用户点击 广播发送 按钮或使用 Ctrl/Cmd + Enter 快捷键
2. 按钮进入"加载中"状态
3. 获取文本内容并分发给所有被选中的 AI视图
4. 清空 Textarea 的内容
5. 当所有已选的 AI视图 都报告发送完成或超时后，按钮恢复到默认态

### 2.6. 组件六: 设置页面 (Settings Page)

**PRD关联:** F-006

**组件描述:**  
一个独立的页面或模态框，用于管理应用级别的配置。

**UI构成:**

**AI模型管理:**
- 一个列表，显示当前已配置的AI
- 每一项包含：AI名称、网址
- 提供 **[按钮] 添加新AI** 和每项的 **[按钮] 编辑/删除**

**缓存管理:**
- **[按钮] 一键清除所有缓存:** 用于清除所有AI的Cookies和登录信息

**其他:** 关于信息、版本号等

**功能逻辑与交互:**
- **清除缓存:** 点击按钮后，弹出一个二次确认对话框，确认后执行清除操作
- **管理AI列表:** 用户在此处进行的增删改操作，会实时反映在 工作区 的AI视图布局和 全局输入栏 的选择器列表上

### 2.7. 组件七: 添加AI对话框 (Add AI Dialog)

**PRD关联:** F-001, F-006

**组件描述:**  
一个模态对话框，作为用户向工作区快速添加AI服务的主要入口。

**UI构成:**
- **对话框标题:** "添加AI助手"
- **预设AI列表:**
  - 一个网格或列表，展示预设的、主流的AI服务（如 Gemini, DeepSeek, Kimi, ChatGPT 等）
  - 每一项包含该AI的官方Logo和名称
- **操作按钮:**
  - **[按钮] 添加:** 当用户选择至少一个AI后，此按钮变为可用状态
  - **[按钮] 取消** 或 **关闭图标 (x):** 关闭对话框

**功能逻辑与交互:**

**触发:** 用户点击 工作区 中的 添加AI视图 (+) 按钮后，该对话框弹出

**选择:**
- 用户可以点击一个或多个预设的AI项进行选择
- 已存在于工作区中的AI项应被禁用或标记为"已添加"

**确认添加:**
1. 用户点击 添加 按钮
2. 对话框关闭
3. 所有被选中的AI服务将作为新的 AI视图 被添加到 工作区
4. 新添加的AI配置会被持久化保存

### 2.8. 组件八: 首次启动与引导流程 (Onboarding Flow) (V1.2 新增)

**PRD关联:** F-001, F-006

**组件描述:**  
为首次打开应用的用户提供的一套引导流程，旨在帮助用户快速完成初始设置并理解核心功能。

**触发条件:**
- 应用在本地存储中检测不到任何用户配置时自动触发

**流程步骤:**

**1. 欢迎弹窗:**
- **UI:** 一个简洁的模态对话框，包含欢迎语（如"欢迎使用 ParallelChat！一个窗口，多AI协作。"）和核心价值说明
- **操作:** 提供一个明确的 **[开始配置]** 按钮

**2. 强制引导添加AI:**
- **UI:** 点击 **[开始配置]** 后，自动弹出 添加AI对话框
- **逻辑:** 此为进入应用的前置步骤，添加 按钮在用户至少选择一个AI之前为禁用状态

**3. 进入主界面与核心功能提示:**
- **逻辑:** 用户完成AI添加后，进入主界面
- **UI:** 出现一个一次性的高亮提示（Coach Mark），箭头指向 全局输入栏
- **提示内容:** "就是这里！输入你的问题，按下 Ctrl+Enter 即可一次性发送给所有选中的AI！"
- **交互:** 用户点击提示或在输入框输入内容后，该提示自动消失
